# site-mgsport-pages

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面通常用于展示、测试、演示或临时部署场景，不针对任何特定域名或具体网站。

项目旨在提供一个结构化的页面存放空间，方便不同来源的静态页面统一管理、版本追踪与历史回溯。

## 目录说明

```
site-mgsport-pages/
├── pages/          # 存放所有 HTML 页面及其相关资源
├── assets/         # 公共资源文件（如 CSS、JavaScript、图片等）
├── docs/           # 项目文档或页面说明
└── README.md       # 本文件
```

- `pages/`：每个子目录对应一个独立页面，包含 `.html` 文件及其附属资源。
- `assets/`：多个页面可共享的公共资源，按类型或用途组织。
- `docs/`：用于存放页面使用说明、设计稿或变更记录等文档。

## 页面归档说明

本仓库中的页面遵循以下归档原则：

- 每个 HTML 页面应尽可能自包含，避免对外部资源的强依赖。
- 页面命名清晰，避免重复或含义模糊的名称。
- 如有必要，可在页面目录内添加 `README.md` 说明该页面的用途、依赖或注意事项。
- 历史版本通过 Git 提交记录保留，不建议在仓库内保留多个版本副本。

## 维护说明

- 提交新页面时，请先检查是否存在相似页面，避免冗余。
- 更新已有页面时，请在提交信息中简要说明变更内容。
- 若页面不再需要，建议在提交信息中说明删除原因，并确保无其他页面依赖该资源。
- 公共资源（如样式、脚本）的修改需确认不影响其他页面正常使用。

## 许可

本仓库内容按需使用，不设特定许可证。如需引用或分发，请自行评估合规性。
